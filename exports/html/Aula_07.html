O que uma Estrutura de deciso?

um bloco de programao que divide o seu cdigo em dois algoritmos distintos, mas somente um deles ser executado com base em uma condio determinada pelo prprio programador: caso a condio seja verdadeira, ele ir executar um algoritmo. Caso essa mesma condio seja falsa, um outro algoritmo ser executado no lugar. Dessa forma, o programa poder ser finalizado de duas formas diferentes. Ambos os algoritmos sero definidos pelo programador. H tambm a possibilidade de ter mais de dois caminhos diferentes, e pode acontecer ainda do computador precisar avaliar duas ou mais condies diferentes simultaneamente. Entenda que a partir daqui que iremos precisar dos operadores relacionais vistos na aula passada.

Estrutura if...else

De todas as estruturas que existem em programao, a mais famosa (e usada) , de longe, o if...else, tambm conhecido como se...seno. Se voc usa o Excel, ento deve estar familiarizado com essa estrutura: ela usada na funo =SE(condio;valor_se_verdadeiro;valor_se_falso) do Excel, e segue a mesma linha de raciocnio.

Funciona assim: suponha que um programa precisa verificar se um determinado usurio de nome Fulano e idade 20 seja maior de idade ou no. Ento o dev ir desenvolver a seguinte estrutura:

In[]:
# variveis
nome = 'Fulano'
idade = 20

# estrutura de deciso
if idade >= 18:
    print(f'{nome}  maior de idade.')
else:
    print(f'{nome}  menor de idade.')

Fulano  maior de idade.

No caso acima, o programa ir executar apenas a primeira sada de dados, e ignorar a segunda, j que ela satisfez a condio imposta. Agora vamos trocar a idade do Fulano para 15 e rodar de novo o programa:

In[]:
# variveis
nome = 'Fulano'
idade = 15

# estrutura de deciso
if idade >= 18:
    print(f'{nome}  maior de idade.')
else:
    print(f'{nome}  menor de idade.')

Fulano  menor de idade.

Veja que agora o resultado final foi diferente: ele ignorou a primeira sada de dados, e executou a segunda, j que a condio imposta no foi atendida.

Indentao

Indentao de cdigo uma prtica que consiste em organizar o seu cdigo para deix-lo mais organizado, e consequentemente, mais legvel e mais fcil para manuteno. A tcnica adicionar um espaamento, ou um recuo esquerda das linhas de comando que ficam dentro das estruturas, e dessa forma podemos identificar mais facilmente o que est dentro de um escopo e o que est fora dele. O problema que essa uma prtica que no seguida por todos, o que acaba resultando em cdigos confusos e problemticos na hora de dar manuteno.

Python resolveu muito bem esse problema, pois nessa linguagem em questo, a indentao obrigatria, j que ela que identifica se um trecho de cdigo est ou no dentro de uma estrutura, diferentemente das outras linguagens, onde o que define os limites da estrutura so as chaves ({}). Em outras palavras: deixe de indentar o seu cdigo Python ou indente incorretamente, e garantido que seu cdigo dar errado. Veja o exemplo abaixo:

if idade >= 18:
4x espao ou 1x tab print(f'{nome} maior de idade.')

4x espao ou 1x tab print(f'{nome} menor de idade.')

O comando elif

Foi falado mais cedo que possvel encadear mais do que dois caminhos em uma estrutura de if...else. Caso isso seja necessrio, enquanto que em outras linguagens se utiliza o comando else if, em Python ele substitudo por elif. Vejamos um exemplo prtico: digamos que um aluno chamado Fulano tirou mdia 10 na nota final da escola, o Cicrano tirou 6, e o Beltrano tirou 4. Nessa escola, a mdia para aprovao 7, mas o aluno s vai para recuperao caso a nota dele tenha ficado entre 5 e 7. Abaixo disso, ele sumariamente reprovado. Ento o sistema vai avaliar primeiro a nota do Fulano em um algoritmo parecido com esse:

In[]:
nome = 'Fulano'
nota = 10

if nota >= 7:
    print(f'{nome} foi aprovado.')
elif nota >= 5:
    print(f'{nome} est de recuperao.')
else:
    print(f'{nome} est reprovado.')

Fulano foi aprovado.

Vejam que o Fulano foi aprovado, pois sua nota era maior que a mnima necessria para aprovao. Vejamos agora a nota do Cicrano:

In[]:
nome = 'Cicrano'
nota = 6

if nota >= 7:
    print(f'{nome} foi aprovado.')
elif nota >= 5:
    print(f'{nome} est de recuperao.')
else:
    print(f'{nome} est reprovado.')

Cicrano est de recuperao.

Cicrano acabou ficando de recuperao, pois sua nota era menor que a mnima para aprovao, porm era maior que o limite da reprovao. isso que o elif faz: adiciona uma segunda verificao caso a primeira seja falsa. Dessa forma, se o meu cdigo entrar dentro da verificao do elif, porque com certeza sua nota menor que 7, logo no ter risco de conflitos. Agora, vamos verificar a nota do Beltrano:

    print(f'{nome} foi aprovado.')

else:
    print(f'{nome} est reprovado.')

Beltrano est reprovado.

Beltrano no conseguiu sequer atingir a nota mnima para ao menos conseguir a recuperao. Logo, a nota dele entrou na ltima verificao: aquela "dos que sobram".

Estrutura match...case

A estrutura match...case foi introduzida em verses recentes do Python, mais precisamente a 3.10.

Quem j tem experincia com programao em outra linguagem deve se lembrar que em outras linguagens de programao temos uma outra estrutura de deciso muito parecida com a if...else, que a switch...case. Acontece que o Python NO TEM SWITCH...CASE. Ou pelo menos no tinha at a atualizao 3.10. Antes dela, quem quisesse usar um switch...case da vida, teria que apelar para o if...elif...else tradicional ou usar alguma funo maluca. Mas no Python 3.10 foi introduzido o match...case, que possui exatamente a mesma funcionalidade do switch...case.

T, mais e da? Eu no sei o que switch...case nem o que match...case

O match...case, que o equivalente ao switch...case nas outras linguagens, uma estrutura de deciso que permite adicionar vrios caminhos diferentes para o algoritmo (muito mais do que apenas 2) e que permite avaliar valores bem especficos para decidir qual caminho tomar. Ele acaba sendo um algoritmo mais ideal para alguns casos especficos, como por exemplo, quando eu tiver vrias possibilidades de caminhos diferentes, ou quando as comparaes a serem feitas so valores absolutos, ao invs de intervalos de nmeros. Aqui, ele no compara usando os operadores relaccionais, mas sim verifica valor por valor, um por um, e retorna um nico valor quando ele no conseguir encontrar um valor exato.

Vamos para um exemplo prtico: eu vou para o cinema, mas antes de escolher o filme, gostaria de saber qual filme est passando em qual sala. Digamos que o cinema em questo tem 5 salas. Se eu selecionar um nmero dentro do nmero de salas existentes, o programa dever me retornar o nome do filme que est passando na sala escolhida. Segue o algoritmo abaixo:

In[]:
# entrada de dados
sala = int(input('Informe o nmero da sala de 1 a 5:'))

# verifica a sala
match sala:
    case 1:
        print('Filme: A Volta dos Que No Foram.')
    case 2:
        print('Filme A Roda Quadrada.')
    case 3:
        print('Filme Poeira em Alto Mar.')
    case 4:
        print('Filme As Tranas do Rei Careca.')
    case 5:
        print('Filme A Vingana do Peixe Frito.')
    case _:
        print('Sala inexistente.')

Filme Poeira em Alto Mar.

O match faz o papel do comando switch das outras linguagens. A ltima opo, que nas outras linguagens era chamado de default, aqui usamos o case _: no lugar. Repare tambm que no usamos o comando break, usado para interromper um case. O comando break at existe no Python, mas no utilizado aqui, e tambm no para essa finalidade (veremos sobre ele nas aulas futuras). Aqui simplesmente no h a necessidade de colocar o comando break, nem de substitu-lo, j que o break aqui feito de forma automtica.

Estrutura try...except

Para os que j tm experincia com programao, a estrutura do try...except a mesma do try...catch das outras linguagens de programao tradicionais. Para os que esto vendo programao pela primeira vez, o try...except uma estrutura de deciso usada para tratamento de excees. Ela funciona de forma parecida com a estrutura if...else, com a diferena de que ele ir "tentar" executar o bloco do try, independente de condio ou no, mas caso esse bloco, por algum motivo, der algum erro, ele para e executa outro algoritmo, o do except. Esse algoritmo muito usado para evitar que o usurio veja mensagens muito tcnicas de erros no programa, e retorne uma mensagem de erro mais amigvel no lugar. Segue abaixo um exemplo:

In[]:
valor1 = '10' # string
valor2 = 10 # int

try:
    print(valor1 + valor2)
except:
    print('No foi possvel rodar o programa. Desculpe!')

No foi possvel rodar o programa. Desculpe!

No caso acima, o programa retorna a mensagem do except, pois no foi possvel executar o try, visto que ele est tentando somar uma string com um nmero inteiro, o que impossvel para o Python. O algoritmo de baixo o mesmo algoritmo, porm adicionado uma nova instruo para exibir uma mensagem independente de erro no programa ou no:

In[]:
valor1 = '10' # string
valor2 = 10 # int

try:
    print(valor1 + valor2)
except:
    print('No foi possvel rodar o programa. Desculpe!')
finally:
    print('Tente mudar o valor da varivel "valor1" para um nmero inteiro caso d erro.')

No foi possvel rodar o programa. Desculpe!
Tente mudar o valor da varivel "valor1" para um nmero inteiro caso d erro.

Nesse novo algoritmo, ele retorna o erro junto com uma sugesto para que o programa d certo. Isso acontece porque a mensagem da sugesto foi programada dentro da estrutura finally. Essa estrutura executa seu algoritmo independentemente do algoritmo do try ter dado certo ou no.
